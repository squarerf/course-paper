# 课程论文自动生成

根据用户需求，自动生成完整的课程论文Word文档。

## 进度跟踪

论文生成涉及9个步骤（环境检测→需求收集→文献检索→文献验证→内容生成→图表→Word生成→检查→交付）。每步完成后向用户报告进度，等待确认再继续。参考 `references/progress-templates.md` 的模板格式。

## 第零步：环境检测

运行 `scripts/check_env.py` 检测环境，根据输出处理问题：

```bash
python scripts/check_env.py  # 或 python3 / py
```

**为什么需要这一步：** 环境问题（Python不可用、依赖缺失）会导致后续步骤静默失败或产生无意义的错误信息。提前检测可以快速定位问题，避免浪费时间。

**常见问题及修复：**
- `Python 未安装` → `winget install Python.Python.3.12`
- `Node.js 未安装` → `winget install OpenJS.NodeJS.LTS`
- `Python 依赖缺失` → `pip install matplotlib numpy`
- `Node.js 依赖缺失` → `cd scripts && npm install`

**Windows 特有问题：** `python` 命令可能指向 0 字节的 Windows Store stub，脚本会自动检测并跳过。

检测通过后：读取 `references/format-spec.md` 获取默认格式规范。

## 第一步：收集需求信息

向用户收集以下信息（缺失时询问）：

| 项目 | 必填 | 示例 |
|------|------|------|
| 论文题目/方向 | ✅ | 硅基负极材料 |
| 作者信息 | ✅ | 姓名、学号、班级、院系 |
| 字数要求 | ✅ | 6500字 |
| 参考文献数量 | ✅ | ≥20篇 |
| 文献年份要求 | ✅ | 2022-2025年约10篇 |
| 论文结构 | 可选 | 默认：摘要→目录→正文→参考文献 |
| 封面模板 | 可选 | .docx封面文件路径 |
| 格式要求 | 可选 | 无特殊要求则使用默认格式 |

**完成后**：读取 `references/format-spec.md` 应用默认格式规范。

## 第二步：文献检索

使用多个工具联合检索，确保文献真实有效。具体工具用法和检索策略见 `references/search-strategy.md`。

**检索流程概要：**
1. Semantic Scholar 高精度搜索（5-8个关键词组合）→ 收集50-80篇
2. Semantic Scholar 扩展搜索（追溯高引论文参考文献）→ 补充10-15篇
3. WebSearch 补充搜索（中文综述和学位论文）→ 补充5-10篇

**筛选标准：**
- 引用量 > 50（>100为佳）
- 近3年文献占50%以上
- 每个子方向至少1篇

## 第三步：文献验证

**为什么要在写正文前验证文献：** 编造的文献或 DOI 在查重、答辩时会被发现，导致论文被退回甚至学术处分。预先验证可以确保所有引用都是真实可查的，这是学术诚信的基本要求。

**验证流程：**
1. 提取每篇文献的标题、第一作者姓氏、年份
2. 调用 `mcp__doi-check__batchVerifyCitations` 批量验证
3. 对于验证失败的文献：
   - 有 DOI 的：用 `mcp__doi-check__verifyCitation` 单独验证
   - 无 DOI 的：用 `mcp__doi-check__findVerifiedPapers` 搜索替代文献
4. 删除所有无法验证的文献
5. 只保留验证通过的文献用于后续写作

**验证结果处理：**
- ✅ verified + high confidence → 保留
- ⚠️ verified + medium confidence → 检查匹配是否正确
- ❌ not verified → 删除，替换新文献

**为什么不能编造 DOI：** DOI 是文献的唯一标识符，编造的 DOI 无法通过学术数据库验证，会被识别为虚假引用。如果不确定某篇文献的 DOI，用 `findVerifiedPapers` 搜索真实文献。

## 第四步：论文内容生成

### 标准结构

```
1. 封面（分页）
2. 摘要（中文 + 英文）（中英文连续，Keywords后分页）
3. 目录（分页）
4. 正文（各章连续，不分页）
   ├── 1 引言
   ├── 2 分类/理论基础
   ├── 3 关键挑战/问题
   ├── 4 改性策略/解决方案
   ├── 5 产业化/应用前景
   └── 6 总结与展望
5. 参考文献（分页）
```

### 引用编排规则（核心！）

| 规则 | 说明 | 检查方法 |
|------|------|----------|
| **每篇文献至少被引用1次** | N篇文献对应正文N个引用点，遗漏引用会让审稿人质疑文献选择的合理性 | 统计正文[1]-[N]是否齐全 |
| **避免重复引用** | 每个引用点只用一个文献编号，重复引用会让读者困惑并显得不专业 | 检查是否有[1,2]或重复[1] |
| **引用顺序排列** | 参考文献按首次出现顺序排列，这是大多数学术期刊的规范要求 | 正文先出现[1]则排第一 |
| **引用与文献一一对应** | 正文[N]对应参考文献第N条 | 交叉核对 |
| **每章节约1000-1200字** | 确保总字数达标 | 统计中文字符数 |

**引用示例：**
```
正确：硅的理论比容量高达4200 mAh/g[1]...碳包覆可以提高导电性[2]
错误：硅的理论比容量高[1]...碳包覆有效[1]（重复引用）
错误：先提到[5]，后提到[3]（应按出现顺序编号）
```

## 第五步：图表生成

**为什么要在生成前询问用户：** 不同论文对图表的需求差异很大——综述论文需要趋势图，实验论文需要对比图，理论论文可能只需要流程图。提前确认可以避免生成不需要的图表，节省时间和token。

**询问内容：**
1. 图表数量（默认5张 / 3张精简 / 自定义）
2. 图表类型（柱状图、雷达图、流程图、折线图、面积图、饼图等）
3. 图表风格（学术风格 / 彩色活泼 / 灰度）

使用脚本 `scripts/generate_charts.py` 生成图表。调用方式见 `references/code-templates.md`。

**执行流程：**
1. 根据用户选择生成 `charts.json`（含图表数据）
2. 运行 `python scripts/generate_charts.py --input charts.json --output ./images/`
3. 脚本支持11种图表类型：柱状图、雷达图、流程图、折线图、面积图、饼图、散点图、热力图、箱线图、堆叠柱状图、小提琴图

**图表插入规则：**
1. 正文中添加"如图X所示"的描述文字
2. 图片按在正文中出现的顺序命名为图1→图N
3. 图片插入在引用该图的段落**之前**

## 第六步：Word文档生成

使用脚本 `scripts/generate_docx.js` 生成。调用方式见 `references/code-templates.md`。

**执行流程：**
1. 将论文内容整理为 `content.json` 格式
2. 运行 `node scripts/generate_docx.js --input content.json --output 论文方向.docx`
3. 脚本自动生成完整 Word 文档（含封面、摘要、目录、正文、参考文献）

**content.json 中的图片字段格式：**
```json
{
  "images": {
    "fig1": { "path": "./images/fig1.png", "width": 480, "height": 320 },
    "fig2": { "path": "./images/fig2.png", "width": 480, "height": 320 }
  }
}
```
- `path`：相对于 content.json 文件的路径
- `width`/`height`：图片尺寸（像素），用于 Word 中的缩放

## 第七步：输出检查

| 检查项 | 验证方法 | 通过标准 |
|--------|----------|----------|
| 字数达标 | 统计中文字符数 | ≥要求字数 |
| 文献数量 | 统计参考文献条目 | ≥要求数量 |
| 文献真实性 | DOI批量验证通过 | 100%通过 |
| 文献全覆盖引用 | 统计正文引用点 | 每篇文献至少引用1次 |
| 无重复引用 | 检查正文引用标记 | 无重复编号 |
| 引用顺序 | 检查参考文献排列 | 按首次出现顺序 |
| 格式正确 | 标题1/标题2/正文样式 | 符合默认模板 |
| 目录可更新 | 右键目录→更新域 | 自动生成 |
| 图表清晰 | 300dpi以上 | 无模糊 |
| 引用对应 | 正文[1]与参考文献[1]匹配 | 一一对应 |

## 第八步：交付

**最终输出：** `论文方向.docx`（含图表，约1MB）

**告知用户：**
1. 打开Word文档
2. 右键目录区域
3. 选择"更新域" → "更新整个目录"
4. 目录会自动根据标题样式生成

## 异常处理

| 情况 | 处理方式 |
|------|----------|
| 文献数量不足 | 降低筛选标准（引用>30），或扩展关键词组合 |
| 字数不达标 | 增加每章字数至1200-1500字，或添加新小节 |
| Python 不可用 | 提示用户安装 Python，或跳过图表生成 |
| 封面模板读取失败 | 使用默认封面，提示用户手动替换 |
| DOI 验证失败 | 用 `findVerifiedPapers` 搜索替代文献 |
| 引用顺序错误 | 重新排序参考文献，按首次出现排列 |
